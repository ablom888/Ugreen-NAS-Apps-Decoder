# Your own localization of UGOS apps without re-signing

**🌐 Language:** **English** · [中文](../zh/LOCALIZATION.md) · [हिन्दी](../hi/LOCALIZATION.md) · [Español](../es/LOCALIZATION.md) · [Français](../fr/LOCALIZATION.md) · [Русский](../LOCALIZATION.md)

Signature verification in UGOS is performed **only at install time** of the `.upk`
package (more details — [SIGNING.md](SIGNING.md)). After installation, the application runs from
a folder on disk, so your own language is added **by editing the files of the installed
application** — without repacking or re-signing. You only need to update the integrity
manifest `.check-app` so that the runtime `reconcile` check passes.

The tools (Python 3, no dependencies) are in [`tools/localization/`](../tools/localization):

- **`ug_localize.py`** — find the application, create a locale template, apply a translation.
- **`ug_checkapp.py`** — verify/recompute `.check-app`.

## Where the localization is stored

```
<app_dir>/
├── config.json            → languageList field (list of language codes)
├── www/locale/<lang>.json     web interface texts
├── www/locale/<lang>.json.gz  compressed copy (served by nginx) — regenerated by the script
└── i18n/msg.csv               server-side strings (optional)
```

`<lang>` — a code in the UGOS style: `en-US`, `de-DE`, `ru-RU`, `zh-CN`, and so on.

## Quick start (on the NAS itself, as root/admin)

```bash
# 0) copy tools/localization/ to the NAS (both .py files together)

# 1) find the folder of the installed application by appId
python3 ug_localize.py find com.ugreen.cameramgr

# 2) view the current locales and languages
python3 ug_localize.py list com.ugreen.cameramgr

# 3) create a template for a new language from the English locale
python3 ug_localize.py scaffold com.ugreen.cameramgr ru-RU --from en-US

# 4) edit the values (don't change the keys!) in www/locale/ru-RU.json
#    with any editor directly on the NAS

# 5) apply: generates the .gz, adds ru-RU to languageList,
#    updates .check-app and (optionally) restarts the service
python3 ug_localize.py apply com.ugreen.cameramgr ru-RU \
        <app_dir>/www/locale/ru-RU.json --restart
```

Instead of `appId`, in any command you can specify the path to the application folder directly
(if auto-discovery doesn't find it — for example, a non-standard volume).

## Verification and rollback

```bash
# check integrity (all md5 from .check-app)
python3 ug_checkapp.py verify <app_dir>

# apply makes a backup of the previous locale file alongside it: <lang>.json.bak
# rollback: put the .bak back in place and update the manifest again
mv <app_dir>/www/locale/ru-RU.json.bak <app_dir>/www/locale/ru-RU.json
python3 ug_checkapp.py refresh <app_dir> <app_dir>/www/locale/ru-RU.json
```

## What exactly `apply` does

1. Checks that the translation file is valid JSON.
2. Places it into `www/locale/<lang>.json` (or finalizes an already edited one),
   makes a `.bak` backup if the file existed.
3. Generates `www/locale/<lang>.json.gz` (for nginx `gzip_static`).
4. Adds `<lang>` to `config.json → languageList`.
5. Recomputes `.check-app`: updates the md5 of changed files (including
   `config.json`) and registers the new locale files. It doesn't touch files from `config/` —
   they are excluded from the manifest in the original too.
6. With the `--restart` flag, it restarts the application service (`serviceName` from config.json).

## Important notes

- **Edit only the translation values, not the keys** in `<lang>.json` — the keys must
  match the base locale, otherwise the interface will show empty strings.
- Changes survive a restart, but **may be overwritten when the application is updated**
  (a reinstall places the original files). After an update,
  apply the localization again.
- All of this is operations on **your** device. The vendor's signature is not forged and not
  bypassed at install time; we merely supplement the already installed application and
  keep its local integrity manifest in a consistent state.
- You need root/admin access to the NAS filesystem (usually over SSH).
