# बिना दोबारा हस्ताक्षर किए UGOS अनुप्रयोगों की अपनी स्थानीयकरण

**🌐 भाषा:** [English](../en/LOCALIZATION.md) · [中文](../zh/LOCALIZATION.md) · **हिन्दी** · [Español](../es/LOCALIZATION.md) · [Français](../fr/LOCALIZATION.md) · [Русский](../LOCALIZATION.md)

UGOS में हस्ताक्षर की जाँच `.upk` पैकेज के **केवल इंस्टॉलेशन के समय** होती है
(अधिक विवरण — [SIGNING.md](SIGNING.md))। इंस्टॉलेशन के बाद अनुप्रयोग डिस्क की एक
फ़ोल्डर से चलता है, इसलिए अपनी भाषा **इंस्टॉल किए गए अनुप्रयोग की फ़ाइलों को
संपादित करके** जोड़ी जाती है — बिना दोबारा पैकिंग और दोबारा हस्ताक्षर किए। बस अखंडता
मैनिफ़ेस्ट `.check-app` अपडेट करना ज़रूरी है, ताकि रनटाइम-जाँच `reconcile` पास हो जाए।

उपकरण (Python 3, बिना निर्भरताओं के) — [`tools/localization/`](../tools/localization) में:

- **`ug_localize.py`** — अनुप्रयोग ढूँढना, लोकेल की रूपरेखा बनाना, अनुवाद लागू करना।
- **`ug_checkapp.py`** — `.check-app` की जाँच/पुनर्गणना करना।

## स्थानीयकरण कहाँ संग्रहीत होती है

```
<app_dir>/
├── config.json            → поле languageList (список кодов языков)
├── www/locale/<lang>.json     тексты веб-интерфейса
├── www/locale/<lang>.json.gz  сжатая копия (её отдаёт nginx) — регенерируется скриптом
└── i18n/msg.csv               серверные строки (опционально)
```

`<lang>` — UGOS शैली में कोड: `en-US`, `de-DE`, `ru-RU`, `zh-CN` इत्यादि।

## त्वरित शुरुआत (स्वयं NAS पर, root/admin के अंतर्गत)

```bash
# 0) скопировать tools/localization/ на NAS (оба .py рядом)

# 1) найти папку установленного приложения по appId
python3 ug_localize.py find com.ugreen.cameramgr

# 2) посмотреть текущие локали и языки
python3 ug_localize.py list com.ugreen.cameramgr

# 3) создать заготовку для нового языка из английской локали
python3 ug_localize.py scaffold com.ugreen.cameramgr ru-RU --from en-US

# 4) отредактировать значения (ключи не меняем!) в www/locale/ru-RU.json
#    любым редактором прямо на NAS

# 5) применить: генерирует .gz, добавляет ru-RU в languageList,
#    обновляет .check-app и (опц.) перезапускает сервис
python3 ug_localize.py apply com.ugreen.cameramgr ru-RU \
        <app_dir>/www/locale/ru-RU.json --restart
```

किसी भी कमांड में `appId` के बजाय अनुप्रयोग की फ़ोल्डर का पथ सीधे दिया जा सकता है
(यदि स्वतः-खोज नहीं ढूँढ पाती — उदाहरण के लिए, ग़ैर-मानक वॉल्यूम)।

## जाँच और वापसी (रोलबैक)

```bash
# проверить целостность (все md5 из .check-app)
python3 ug_checkapp.py verify <app_dir>

# apply делает бэкап предыдущего файла локали рядом: <lang>.json.bak
# откат: вернуть .bak на место и снова обновить манифест
mv <app_dir>/www/locale/ru-RU.json.bak <app_dir>/www/locale/ru-RU.json
python3 ug_checkapp.py refresh <app_dir> <app_dir>/www/locale/ru-RU.json
```

## `apply` वास्तव में क्या करता है

1. जाँचता है कि अनुवाद की फ़ाइल — वैध JSON है।
2. उसे `www/locale/<lang>.json` में रखता है (या पहले से संपादित को अंतिम रूप देता है),
   यदि फ़ाइल पहले से थी तो `.bak` बैकअप बनाता है।
3. `www/locale/<lang>.json.gz` उत्पन्न करता है (nginx `gzip_static` के लिए)।
4. `config.json → languageList` में `<lang>` जोड़ता है।
5. `.check-app` की पुनर्गणना करता है: बदली हुई फ़ाइलों (`config.json` सहित) के md5
   अपडेट करता है और लोकेल की नई फ़ाइलों को पंजीकृत करता है। `config/` की फ़ाइलों को नहीं छूता —
   वे मूल में भी मैनिफ़ेस्ट से बाहर हैं।
6. `--restart` फ़्लैग पर अनुप्रयोग का सर्विस पुनः प्रारंभ करता है (`serviceName` — config.json से)।

## महत्वपूर्ण टिप्पणियाँ

- **`<lang>.json` में केवल अनुवाद के मान संपादित करें, कुंजियाँ नहीं** — कुंजियाँ आधार
  लोकेल से मेल खानी चाहिए, अन्यथा इंटरफ़ेस खाली स्ट्रिंग्स दिखाएगा।
- बदलाव पुनः प्रारंभ के बाद बने रहते हैं, पर **अनुप्रयोग के अपडेट के समय मिट सकते हैं**
  (पुनः-इंस्टॉलेशन मूल फ़ाइलें रख देता है)। अपडेट के बाद स्थानीयकरण दोबारा लागू करें।
- यह सब — **आपके** डिवाइस पर की जाने वाली क्रियाएँ हैं। इंस्टॉलेशन के समय वेंडर के
  हस्ताक्षर की न नकल की जाती है और न ही उसे दरकिनार किया जाता है; हम केवल पहले से इंस्टॉल
  किए गए अनुप्रयोग को पूरक बनाते हैं और उसके स्थानीय अखंडता मैनिफ़ेस्ट को संगत रखते हैं।
- NAS की फ़ाइल-सिस्टम तक root/admin पहुँच चाहिए (आमतौर पर SSH के माध्यम से)।
