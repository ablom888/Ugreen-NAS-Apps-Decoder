# Своя локализация приложений UGOS без переподписи

**🌐 Язык:** [English](en/LOCALIZATION.md) · [中文](zh/LOCALIZATION.md) · [हिन्दी](hi/LOCALIZATION.md) · [Español](es/LOCALIZATION.md) · [Français](fr/LOCALIZATION.md) · **Русский**

Проверка подписи в UGOS выполняется **только при установке** пакета `.upk`
(подробнее — [SIGNING.md](SIGNING.md)). После установки приложение работает из
папки на диске, поэтому свой язык добавляется **правкой файлов установленного
приложения** — без переупаковки и переподписи. Нужно лишь обновить манифест
целостности `.check-app`, чтобы прошла рантайм-проверка `reconcile`.

Инструменты (Python 3, без зависимостей) — в [`tools/localization/`](../tools/localization):

- **`ug_localize.py`** — найти приложение, создать заготовку локали, применить перевод.
- **`ug_checkapp.py`** — проверить/пересчитать `.check-app`.

## Где хранится локализация

```
<app_dir>/
├── config.json            → поле languageList (список кодов языков)
├── www/locale/<lang>.json     тексты веб-интерфейса
├── www/locale/<lang>.json.gz  сжатая копия (её отдаёт nginx) — регенерируется скриптом
└── i18n/msg.csv               серверные строки (опционально)
```

`<lang>` — код в стиле UGOS: `en-US`, `de-DE`, `ru-RU`, `zh-CN` и т.п.

## Быстрый старт (на самом NAS, под root/admin)

```bash
# 0) скопировать tools/localization/ на NAS (оба .py рядом)

# 1) найти папку установленного приложения по appId
python3 ug_localize.py find com.ugreen.cameramgr

# 2) посмотреть текущие локали и языки
python3 ug_localize.py list com.ugreen.cameramgr

# 3) создать заготовку для нового языка из английской локали
python3 ug_localize.py scaffold com.ugreen.cameramgr ru-RU --from en-US

# 4) отредактировать значения (ключи не меняем!) в www/locale/ru-RU.json
#    любым редактором прямо на NAS

# 5) применить: генерирует .gz, добавляет ru-RU в languageList,
#    обновляет .check-app и (опц.) перезапускает сервис
python3 ug_localize.py apply com.ugreen.cameramgr ru-RU \
        <app_dir>/www/locale/ru-RU.json --restart
```

Вместо `appId` в любой команде можно указать путь к папке приложения напрямую
(если автопоиск не находит — например, нестандартный том).

## Проверка и откат

```bash
# проверить целостность (все md5 из .check-app)
python3 ug_checkapp.py verify <app_dir>

# apply делает бэкап предыдущего файла локали рядом: <lang>.json.bak
# откат: вернуть .bak на место и снова обновить манифест
mv <app_dir>/www/locale/ru-RU.json.bak <app_dir>/www/locale/ru-RU.json
python3 ug_checkapp.py refresh <app_dir> <app_dir>/www/locale/ru-RU.json
```

## Что именно делает `apply`

1. Проверяет, что файл перевода — валидный JSON.
2. Кладёт его в `www/locale/<lang>.json` (или финализирует уже отредактированный),
   делает бэкап `.bak`, если файл существовал.
3. Генерирует `www/locale/<lang>.json.gz` (для nginx `gzip_static`).
4. Добавляет `<lang>` в `config.json → languageList`.
5. Пересчитывает `.check-app`: обновляет md5 изменённых файлов (включая
   `config.json`) и регистрирует новые файлы локали. Файлы из `config/` не трогает —
   они и в оригинале исключены из манифеста.
6. По флагу `--restart` перезапускает сервис приложения (`serviceName` из config.json).

## Важные замечания

- **Правьте только значения переводов, не ключи** в `<lang>.json` — ключи должны
  совпадать с базовой локалью, иначе интерфейс покажет пустые строки.
- Изменения переживают перезапуск, но **могут быть затёрты при обновлении
  приложения** (переустановка кладёт оригинальные файлы). После обновления
  примените локализацию снова.
- Всё это — операции на **вашем** устройстве. Подпись вендора не подделывается и не
  обходится при установке; мы лишь дополняем уже установленное приложение и
  поддерживаем его локальный манифест целостности в согласованном виде.
- Нужен доступ root/admin к файловой системе NAS (обычно по SSH).
